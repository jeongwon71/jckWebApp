Performance Benchmark for jck Applications

Contact: Jeongwon Seo (seo71@purdue.edu)

Objective

One of the most crucial aspects of model validation is conducting experiments that align with simulations of the implemented model/system(s). To reduce the associated costs and infer the additional information that new experiments might contribute to the application, the ACCRUE index was previously proposed. Therefore, the objective of this project is to develop confidence in data assimilation (DA) techniques by employing three distinct sorting schemes. Experiments were sorted in ascending and descending order using the ck metric, while the most recent methodology, jk, was used to achieve optimal sorting. The results obtained from experiments sorted by different methods allow for a comparison between traditional ck-based sorting and the newer jk-based sorting approach.


Description

This exercise is designed to perform DA using deterministic, stochastic, or mixed modes. DA is performed on a single application using available experiments (a single or multiple). If a deterministic mode is chosen, the mean and covariance CSV files are used. The size of the mean and covariance for the simulation information is Ne+1 and Ne+1 by Ne+1, respectively. On the other hand, the size of the mean and covariance for the measurement information is Ne and Ne by Ne, respectively, since the application has no measurement. Additionally, users can select the stochastic mode by specifying the desired number of samples. To obtain statistically stable results, at least 300 samples are recommended, though having more than 1000 would be ideal. When choosing the stochastic mode, a file with a size of Ns by Ne+1 is for simulation information, and a CSV file with a size of Ns by Ne is uploaded for measurement information. (Ne : Number of experiments, Ns : Number of samples)


Expected Results and Deliverables

Once the files are successfully uploaded, participants can download both a PNG file representing the graph of the results and a CSV file containing the data corresponding to the table. The graph and table display the results of the DA process as experiments are added in an incremental manner by three different sorting methods. The data point at index 0 on x-axis of the graph, as well as the first row of the table (excluding the header), where the number of experiments is zero, represents the prior information of the application, i.e., the mean and standard deviation. Since no experiments are used at this point, the results should be identical regardless of the sorting method. Subsequently, as experiments are included in the DA calculation process according to the sorting method, the updated application information is recorded. The final DA result, after all experiments have been utilized, is the same regardless of the sorting method. Hence, the data points at the last index of the graph and the last row of the table are identical across all methods. The downloaded CSV file consists of six columns: odd-numbered columns represent the keff mean, while even-numbered columns represent the keff uncertainty. Columns 1 and 2 show the results from ck-ascending sorting, where experiments are sorted from the smallest to the largest ck value; columns 3 and 4 correspond to ck-descending sorting, the reverse order; and finally, columns 5 and 6 correspond to optimal sorting using jk, as proposed in the ACCURE paper.



Instructions


The process of practicing DA using three different sorting methods on this website can be broadly divided into three main steps:

  1. File preparation
    2. Participants can prepare their data/files using one of the following methods:
    1. Demo files - Information on 10 experiments and an application model from the ICSBEP handbook has been pre-saved. You can download the files by clicking the links in the 'Demo Files' section below.
    2. ICSBEP handbook benchmarks - Participants can select experiments and an application from a total of 425 critical benchmarks. After selecting all benchmarks for both the experiments and the application, click the 'Selection complete' button to choose one application from the selected benchmarks. Once the application is chosen, you can download the simulation keff mean, measurement keff mean, simulation keff covariance, and measurement keff covariance using the four buttons below.
    3. Participants can prepare their own files/data. However, be mindful of the difference in data size between simulation and measurement information depending on whether an application is involved. For more information on file sizes, refer to the 'Description' section above.
  2. Upload files
    3. Participants can determine the appropriate file mode for performing DA based on their available data. Please use the dropdown feature next to 'Select an option' in the 'Upload CSV File' section to choose the mode.
    1. Deterministic mode - To perform DA in this mode, two files are required: one containing the keff mean values and another containing the keff covariance. If both simulation and measurement information are to be uploaded in deterministic mode, a total of four files will be needed. Please ensure that the CSV file containing the mean values is a column vector. Additionally, if the provided covariance is not positive semi-definite, errors may occur during the calculation process.
    2. Stochastic mode - DA can be calculated based on samples, as the keff mean and covariance can also be derived from samples. If you wish to use stochastic mode, upload files containing the samples corresponding to the simulation and measurement information. Each column in the sample files must form a column vector representing the samples for the respective experiment/application. Therefore, the files should be sized Ns by Ne+1 for simulation information or Ns by Ne for measurement information.
    3. Mixed mode - It is possible to use deterministic and stochastic modes interchangeably. Depending on the participant’s preference, simulation information can be uploaded in deterministic mode while measurement information is uploaded in stochastic mode, or vice versa.
  3. Download results
    4. Participants can view the results in both graph and table formats, and download them for further analysis or future research. These functions become available after providing the correct files and clicking the 'Upload and Process' button. Once activated, by clicking the 'Download Graph' and 'Download Data' buttons, the graph will be saved in PNG format, and the table will be provided as a CSV file.

Demo files is provided for practice purpose. The data includes 10 randomly selected experiments and one application from the ICSBEP handbook criticality benchmarks. Both the simulation information and measurement information can be uploaded in either deterministic or stochastic mode (interchangeable). The number of samples used in the stochastic mode for both simulation and measurement modes is 10,000.

  • For Stochastic mode, simulation samples, and measurement samples
